# ⏱ Time-Up
> TimeUp 프론트엔드 레포지토리입니다.

## 🛠️ 기술 스택

- **React Native** 
- **Expo**
- **TypeScript** 
- **NativeWind** – Tailwind

### 📍 브랜치 규칙 (Git-flow 전략)
- **main** : 기준이 되는 브랜치로 제품을 배포하는 브랜치
- **dev** : 개발 브랜치로 이 브랜치를 기준으로 각자 작업한 기능들을 Merge
- **feat** : 단위 기능을 개발하는 브랜치로 기능 개발이 완료되면 develop 브랜치에 Merge

### ✅ 커밋 메시지 규칙

| 태그 | 설명 |
|------|------|
| `feat` | 새로운 기능 추가 |
| `add` | 새로운 파일 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 수정 |
| `style` | 코드 스타일 변경  |
| `refactor` | 기능 변경 없이 코드 개선 |
| `test` | 테스트 코드 추가/수정 |
| `chore` | 빌드, 의존성 등 기타 작업 |
| `hotfix` | 운영 중 긴급 버그 수정 |
